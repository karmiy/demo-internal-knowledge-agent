# 文档预览与本地检索修复设计

## 背景

管理员文档页目前只能展示标题和处理状态，无法查看 Ingest 后真正参与检索的内容。与此同时，`andy.admin` 查询“公司的核心协作时间是什么？”时被安全拒答。诊断结果表明，管理员角色和 ACL 均正常：正确的《员工手册》“工作时间”分段排在检索第一，但 cosine distance 为 `0.5417`，超过 Agent 当前沿用的 `0.45` 阈值。

该阈值早于本地哈希 Embedding 引入。现有索引又只嵌入正文，没有把文档标题和章节加入向量，因此“工程发布流程”等依赖标题或章节措辞的问题也容易被误拒答。

## 目标

- 管理员可以从文档列表打开右侧抽屉，预览 Ingest 后的章节和正文分段。
- 文档详情展示状态、访问范围和分段数量，但不暴露服务器文件路径。
- 修复已授权且有明确证据的问题被本地 Embedding 阈值误拒答的问题。
- 已经入库的旧索引可以自动升级，无需用户删除数据库卷。
- 聊天输入框隐藏可见的“问题”标签，并支持 Enter 发送、Shift+Enter 换行。

## 非目标

- 不实现 PDF 浏览器、DOCX 渲染器或原始文件下载。
- 不开放普通用户读取完整文档内容的管理 API。
- 不更换当前本地 Embedding，也不引入新的模型供应商。
- 不改变薪资工具或文档 ACL 的授权规则。

## 文档详情 API

新增 `GET /api/admin/documents/{document_id}`，继续复用现有管理员依赖：

- 非管理员返回 `403`。
- 文档不存在返回 `404`。
- 成功响应包含文档 ID、标题、状态、错误信息、创建及更新时间、权限列表和分段列表。
- 权限项包含 `subject_type` 与 `subject_id`。
- 分段按 `chunk_index` 升序排列，包含章节、页码、正文和索引序号。
- 不返回 `source_path`、checksum 或 embedding 数组。

列表 API 保持轻量，不在列表响应中携带所有正文。前端只有在用户点击一行时才请求详情。

## 文档预览交互

管理员点击文档行后，从页面右侧打开详情抽屉，列表页仍保留在背景中：

- 抽屉头部展示标题、处理状态和关闭按钮。
- 元信息区展示访问范围和分段数量。
- 正文区按 Ingest 顺序展示章节、页码和分段文本。
- 加载过程中显示明确的加载状态。
- 请求失败显示错误信息，并允许关闭后重试。
- 文档尚未产生分段时显示“文档尚未完成处理”。
- 点击关闭按钮或按 Escape 关闭抽屉；打开时使用语义化 dialog 属性。
- 文档行具备按钮语义、键盘焦点和清晰的 hover/focus 状态。

## 本地检索修复

### 根因修复

Ingest 生成向量时，将以下内容组合后交给本地 Embedder：

1. 文档标题；
2. 分段章节；
3. 分段正文。

标题和章节只用于生成向量，不重复写入用户看到的正文。这样“工程发布流程”能够与《工程团队指南》/“发布流程”建立直接的词法特征重合。

本地哈希 Embedding 的接受距离使用配置项 `RETRIEVAL_MAX_DISTANCE`，默认值校准为 `0.72`，不再继续复用旧语义 Embedding 的 `0.45`。API 创建 Agent 时显式传入该配置。验收数据必须同时证明：

- “公司的核心协作时间是什么？”接受《员工手册》“工作时间”；
- “工程发布流程是什么？”接受《工程团队指南》“发布流程”；
- 明显无关的问题仍安全拒答。

### 自动重新索引

每个分段的 metadata 写入索引版本 `local-hash-v2`。Ingest Worker 启动后除处理 `PENDING` 文档外，也查找 `READY` 但缺少该版本的文档，并通过现有幂等 Ingest 流程重新生成分段和向量。

重新索引失败时沿用现有失败状态和错误信息；成功后恢复为 `READY`。这保证已有 Docker volume 可以平滑升级，不要求执行 `docker compose down -v`。

ACL 仍然在 SQL `WHERE` 中先执行，再做 pgvector 排序和 `LIMIT`。管理员全量访问规则不变。

## 聊天输入交互

- 删除输入框上方可见的“问题”标签。
- textarea 保留 `aria-label="问题"`，不降低可访问性和测试可定位性。
- Enter 在输入非空且非请求中时提交表单，并阻止插入换行。
- Shift+Enter 使用 textarea 默认行为插入换行。
- 输入法处于 composition 状态时，Enter 只用于确认候选词，不提交表单。
- 原有发送按钮和禁用逻辑保持不变。

## 测试与验收

### 后端

- 管理员可以读取文档详情，分段顺序和权限信息正确。
- 普通用户读取详情返回 `403`，不存在文档返回 `404`。
- 向量输入包含标题、章节和正文。
- Worker 会选择旧索引版本的 READY 文档，并在成功后写入新版本。
- 预置的核心协作时间与工程发布问题通过阈值，明显无关问题仍拒答。

### 前端

- 点击文档行请求并展示详情抽屉。
- 抽屉展示权限、章节和正文，并支持关闭、加载、错误与空分段状态。
- 页面不再显示输入框标签“问题”，但 textarea 仍有可访问名称。
- Enter 调用发送，Shift+Enter 保留换行，composition Enter 不发送。

### 端到端

重新构建并启动 Compose 后，确认旧的三份文档自动完成新版本索引。使用 `andy.admin` 提问“公司的核心协作时间是什么？”，应返回工作日 `10:00` 至 `16:00`，并引用《员工手册》“工作时间”。文档管理页应可打开三份预置文档的详情抽屉。
