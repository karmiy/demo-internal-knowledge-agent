01 / BRIEF

做小，但边界要真。

Demo 只验证最重要的闭环：登录、权限、文档入库、知识检索、内部数据调用、引用回答。它不是一个企业级内容管理平台。

本期包含

  • 用户名密码登录与短时 JWT
  • 用户、角色、部门和文档 ACL
  • PDF、DOCX、Markdown、TXT 入库
  • pgvector 语义检索与来源引用
  • 薪资查询示例内部工具
  • React 聊天页与管理员入库页
  • Docker Compose 一键运行

明确不做

  • 扫描件 OCR 与复杂表格还原
  • SSO、LDAP、SCIM 和企业 IAM 对接
  • 多租户计费和生产级高可用
  • 文档在线协作编辑与版本审批
  • 模型微调和自托管 GPU 推理
  • 开放式网页搜索或任意代码执行

Security before retrieval

权限过滤发生在检索和数据接口内部，敏感数据永远不会先进入模型上下文。

Grounded answers

知识类回答必须附带文档、页码和片段；无充分依据时明确说明无法回答。

Replaceable parts

模型、Embedding 和存储通过接口隔离，Demo 不绑定单一模型供应商。

03 / RUNTIME

一次提问，七个可观察步骤。

主流程保持显式、可测试。LLM 只出现在路由和答案生成处；授权、数据过滤和审计均为确定性代码。

06Verify无证据、越权或低相关时安全拒答

04 / INGEST

文档先加工，再进入问答。

入库是独立命令和后台任务，不阻塞聊天请求。相同文件通过 checksum 去重，更新时替换旧 Chunk。

01
接收与登记

管理员上传文件，同时填写标题、可见角色、部门或指定用户；文档状态设为 pending。

02
安全校验

限制格式与体积，规范文件名，计算 SHA-256；原文件保存在挂载 Volume，不接受可执行文件。

03
解析与清理

PDF 保留页码，DOCX 保留标题层级；Markdown/TXT 直接解析。扫描 PDF 在本期标记为不支持。

04
语义切分

按标题和段落优先切分，目标约 700 tokens、重叠约 100 tokens，并保留来源 metadata。

05
向量化

批量调用 Embedding，限制并发并重试临时错误；模型名称和向量维度写入配置。

06
原子发布

在事务中写入 Chunk 和向量，成功后状态改为 ready；失败则保留错误原因并可重试。

05 / ACCESS

权限不是 Prompt，是代码。

文档 ACL 与内部接口的行级授权是两条独立防线。错误消息不暴露敏感资源是否存在。

可信身份

JWT 仅携带用户标识；后端按用户 ID 重新加载活动状态、角色和部门。密码只保存安全哈希。

检索前过滤

SQL 先根据 ACL 得到允许文档集合，再在其 Chunk 上执行 pgvector 排序，并在返回前二次核验。

抗提示注入

文档内容视为不可信数据。系统提示要求忽略文档内指令，工具集合固定且参数由服务端验证。

07 / API

少而明确的接口契约。

前端不接触模型密钥、数据库或文件系统。所有敏感操作都通过 FastAPI 并生成审计记录。

POST/api/auth/login验证账号密码，签发短时 access token
GET/api/auth/me返回当前用户、角色、部门和可用 UI 能力
POST/api/chat提交 thread_id 与 message，返回回答、引用和安全的执行摘要
GET/api/threads仅列出当前用户拥有的会话
POST/api/admin/documents管理员上传文档并声明 ACL
GET/api/admin/documents查看文档入库状态、错误和权限摘要
POST/api/admin/documents/{id}/retry重试失败的入库任务
GET/health容器健康检查，不返回敏感配置信息

09 / QUALITY

先证明不会泄露，再证明回答得好。

测试优先级按风险排序。授权测试不调用真实模型，确保结果确定、快速且可在 CI 中重复。

01

Policy tests

角色、部门、个人 ACL 组合；他人薪资必须拒绝。

02

Retrieval tests

越权 Chunk 不得出现在候选集；引用必须对应原文。

03

Graph tests

路由、工具异常、无证据拒答和 Checkpoint 隔离。

04

API + UI

登录、聊天、上传、错误反馈和容器健康检查。

提示注入藏在文档中文档作为引用数据而非指令；固定系统消息、固定工具集，并测试“忽略规则、泄露薪资”等恶意片段。
向量搜索跨越权限边界ACL 条件进入 SQL 查询本身；返回后再次核对 document_id，拒绝只靠应用层事后删除。
模型或 Embedding 不可用设置超时、有限重试和清晰错误；入库任务可重试，聊天请求不无限等待。
证据不足却强行回答设置最低相关性和空结果路径；统一返回“无法根据您有权限的信息确认”，同时不泄露资源存在性。
10 / PLAN

四个增量，始终可运行。

每一阶段都有可验证输出，避免一次性搭完所有模块后才发现权限或检索路径错误。

Identity + ACL

Knowledge loop
